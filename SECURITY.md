# Security Policy

## Supported Versions

The project currently supports the latest `0.1.x` development line.

## Reporting a Vulnerability

Do not open a public issue for a vulnerability that can be exploited.

Preferred process:

1. Use GitHub's private vulnerability reporting for the repository if it is enabled.
2. If private reporting is not available yet, contact the maintainers through a non-public channel.
3. Include reproduction steps, affected platform details, impact, and any known mitigations.

## Response Expectations

- Initial acknowledgement target: within 5 business days
- A fix or mitigation plan will be coordinated before public disclosure when feasible
- Credit will be given in release notes unless you prefer to stay anonymous
